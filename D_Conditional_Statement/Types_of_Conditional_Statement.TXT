# CONDITIONAL STATEMENTS:
->CAN BE DEFINED AS STATEMENTS THAT ALLOW THE PROGRAM TO MAKE DECISIONS BASED ON CERTAIN CONDITIONS.
-> THEY HELP IN CONTROLLING THE FLOW OF THE PROGRAM AND EXECUTING DIFFERENT BLOCKS OF CODE BASED ON SPECIFIC CONDITIONS.


NOTE:
() : 'PARANTHESIS' MEANS THESE BRACKETS() HAVE THE HIGHEST PRECEDENCE IN JAVA.
{} : 'CURLY BRACKETS' MEAN THESE BRACKETS{} ARE USED TO GROUP STATEMENTS TOGETHER INTO A BLOCK.


TYPES OF CONDITIONAL STATEMENTS:

1) IF STATEMENT:
-> THE IF STATEMENT IS USED TO EXECUTE A BLOCK OF CODE IF A SPECIFIED CONDITION IS TRUE.IF THE CONDITION IS FALSE, 
   THE BLOCK OF CODE IS SKIPPED.
-> SYNTAX:
   IF (CONDITION) {
       // CODE TO BE EXECUTED IF CONDITION IS TRUE
   }

2) IF-ELSE STATEMENT:
-> THE IF-ELSE STATEMENT IS USED TO EXECUTE ONE BLOCK OF CODE IF A SPECIFIED CONDITION IS TRUE, AND ANOTHER BLOCK OF
   CODE IF THE CONDITION IS FALSE.
-> SYNTAX:
   IF (CONDITION) {
       // CODE TO BE EXECUTED IF CONDITION IS TRUE
   } ELSE {
       // CODE TO BE EXECUTED IF CONDITION IS FALSE
   }

3) ELSE-IF LADDER:
-> THE ELSE-IF LADDER IS USED TO CHECK MULTIPLE CONDITIONS. IT ALLOWS YOU TO TEST SEVERAL CONDITIONS IN SEQUENCE AND 
   EXECUTE THE BLOCK OF CODE CORRESPONDING TO THE FIRST TRUE CONDITION.
-> SYNTAX:
   IF (CONDITION1) {
       // CODE TO BE EXECUTED IF CONDITION1 IS TRUE
   } ELSE IF (CONDITION2) {
       // CODE TO BE EXECUTED IF CONDITION2 IS TRUE
   } ELSE IF (CONDITION3) {
       // CODE TO BE EXECUTED IF CONDITION3 IS TRUE
   } ELSE {
       // CODE TO BE EXECUTED IF ALL CONDITIONS ARE FALSE
   }

4)TARNARY OPERATOR:
-> THE TERNARY OPERATOR IS A SHORTHAND WAY OF WRITING AN IF-ELSE STATEMENT. IT TAKES THREE OPERANDS: A CONDITION, A VALUE
   TO RETURN IF THE CONDITION IS TRUE, AND A VALUE TO RETURN IF THE CONDITION IS FALSE.
-> SYNTAX:
   CONDITION ? VALUE_IF_TRUE : VALUE_IF_FALSE   

5) SWITCH STATEMENT:
-> THE SWITCH STATEMENT IS USED TO EXECUTE ONE BLOCK OF CODE AMONG MANY OPTIONS BASED ON THE VALUE OF A VARIABLE OR AN
   EXPRESSION. IT IS OFTEN USED AS AN ALTERNATIVE TO MULTIPLE IF-ELSE STATEMENTS.
-> SYNTAX:
   SWITCH (EXPRESSION) {
       CASE VALUE1:
           // CODE TO BE EXECUTED IF EXPRESSION EQUALS VALUE1
           BREAK;
       CASE VALUE2:
           // CODE TO BE EXECUTED IF EXPRESSION EQUALS VALUE2
           BREAK;
       CASE VALUE3:
           // CODE TO BE EXECUTED IF EXPRESSION EQUALS VALUE3
           BREAK;
       DEFAULT:
           // CODE TO BE EXECUTED IF EXPRESSION DOES NOT MATCH ANY CASE
   }


   NOTE: 
   
A) THE BREAK STATEMENT IS USED TO EXIT THE SWITCH BLOCK ONCE A MATCHING CASE IS FOUND AND EXECUTED. IF THE BREAK 
   STATEMENT IS OMITTED, THE PROGRAM WILL CONTINUE TO EXECUTE THE FOLLOWING CASES UNTIL IT ENCOUNTERS A BREAK OR 
   REACHES THE END OF THE SWITCH BLOCK.

B) THE DEFAULT CASE IS OPTIONAL AND CAN BE USED TO SPECIFY A BLOCK OF CODE TO BE EXECUTED IF NONE OF THE CASES
   MATCH THE EXPRESSION. IF THE DEFAULT CASE IS OMITTED AND NO CASES MATCH THE EXPRESSION, THEN NO CODE WILL BE EXECUTED.

C) THE SWITCH STATEMENT CAN ONLY BE USED WITH CERTAIN DATA TYPES, SUCH AS BYTE, SHORT, INT, CHAR, ENUMS, AND STRINGS.
   IT CANNOT BE USED WITH FLOAT, DOUBLE, BOOLEAN, OR LONG DATA TYPES.    

D) THE SWITCH STATEMENT IS OFTEN PREFERRED OVER MULTIPLE IF-ELSE STATEMENTS WHEN THERE ARE MANY CONDITIONS TO CHECK, AS IT CAN
   IMPROVE READABILITY AND MAINTAINABILITY OF THE CODE. HOWEVER, FOR SIMPLE CONDITIONS OR WHEN CHECKING RANGES OF VALUES, 
   IF-ELSE STATEMENTS MAY BE MORE APPROPRIATE.

E) IN JAVA 14 AND LATER VERSIONS, THE SWITCH STATEMENT HAS BEEN ENHANCED TO SUPPORT MORE FLEXIBLE SYNTAX AND FEATURES, 
   SUCH AS THE ABILITY TO RETURN VALUES DIRECTLY FROM CASES AND THE INTRODUCTION OF THE "YIELD" KEYWORD FOR MORE COMPLEX 
   EXPRESSIONS. THIS ENHANCEMENT MAKES THE SWITCH STATEMENT EVEN MORE POWERFUL AND VERSATILE FOR HANDLING CONDITIONAL LOGIC IN JAVA.

F) IT IS IMPORTANT TO CHOOSE THE APPROPRIATE CONDITIONAL STATEMENT BASED ON THE SPECIFIC REQUIREMENTS OF YOUR PROGRAM AND THE NATURE
   OF THE CONDITIONS YOU NEED TO EVALUATE. USING THE RIGHT CONDITIONAL STATEMENT CAN HELP IMPROVE THE CLARITY AND EFFICIENCY OF YOUR
   CODE.    

G) WHEN USING CONDITIONAL STATEMENTS, IT IS ALSO IMPORTANT TO CONSIDER THE PERFORMANCE IMPLICATIONS, ESPECIALLY IF YOU HAVE A LARGE NUMBER
   OF CONDITIONS TO CHECK. IN SOME CASES, USING A SWITCH STATEMENT OR A TERNARY OPERATOR MAY BE MORE EFFICIENT THAN MULTIPLE IF-ELSE STATEMENTS. 
   HOWEVER, THE MOST IMPORTANT FACTOR IS USABILITY AND READABILITY OF THE CODE, SO CHOOSE THE CONDITIONAL STATEMENT THAT BEST SUITS YOUR NEEDS AND 
   MAKES YOUR CODE EASY TO UNDERSTAND.

H) IN ADDITION TO THE BASIC CONDITIONAL STATEMENTS, JAVA ALSO PROVIDES OTHER CONTROL FLOW STATEMENTS SUCH AS LOOPS (FOR, WHILE, DO-WHILE) AND 
   EXCEPTION HANDLING (TRY-CATCH-FINALLY) THAT CAN BE USED IN COMBINATION WITH CONDITIONAL STATEMENTS TO CREATE MORE COMPLEX AND ROBUST PROGRAMS. 
   UNDERSTANDING HOW TO USE THESE CONTROL FLOW STATEMENTS EFFECTIVELY IS ESSENTIAL FOR BECOMING A PROFICIENT JAVA PROGRAMMER.

I) IT IS ALSO WORTH NOTING THAT CONDITIONAL STATEMENTS CAN BE NESTED WITHIN EACH OTHER, ALLOWING FOR MORE COMPLEX DECISION-MAKING PROCESSES. 
   HOWEVER, IT IS IMPORTANT TO AVOID EXCESSIVE NESTING, AS IT CAN MAKE THE CODE HARD TO READ AND MAINTAIN. IN SUCH CASES, CONSIDERING REFACTORING THE CODE
   OR USING FUNCTIONS TO SIMPLIFY THE LOGIC MAY BE BENEFICIAL.
